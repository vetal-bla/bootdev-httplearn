# bootdev-httplearn

> This `README.md` was generated by **gpt-5.3-codex**.

Learning project from the [boot.dev](https://www.boot.dev/) HTTP course.
It is a small Go + PostgreSQL API for a "Chirpy" app (users, login, chirps,
refresh tokens, and a webhook to upgrade users to Chirpy Red).

## What this project does

- Serves a static page at `/app/`
- Provides REST endpoints under `/api/*`
- Uses JWT access tokens and DB-backed refresh tokens
- Stores data in PostgreSQL (schema/migrations in `sql/schema`)

## Run with Devbox

### 1) Enter dev environment

```bash
devbox shell
```

`devbox.json` includes: `postgresql`, `goose`, and `sqlc`.

### 2) Start PostgreSQL in Devbox

Use Devbox services to run PostgreSQL:

```bash
devbox services up
```

Check service status:

```bash
devbox services ls
```

Stop services when you are done:

```bash
devbox services stop
```

### 3) Create database

If you do not have it yet:

```bash
createdb chirpy
```

### 4) Configure environment variables

Create `.env` in project root:

```env
DB_URL="postgres://<user>@localhost:5432/chirpy?sslmode=disable"
PLATFORM="dev"
SECRET="<long-random-secret>"
POLKA_KEY="<webhook-api-key>"
```

### 5) Run migrations

```bash
goose -dir sql/schema postgres "$DB_URL" up
```

### 6) Start server

```bash
go run .
```

Server listens on `http://localhost:8080`.

## API quick usage

### Health check

```bash
curl -i http://localhost:8080/api/healthz
```

### Register and login

```bash
curl -s -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"pass123"}'

curl -s -X POST http://localhost:8080/api/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"pass123"}'
```

Login returns:

- `token` (JWT access token)
- `refresh_token`

### Create and read chirps

```bash
# create chirp (Bearer = access token)
curl -s -X POST http://localhost:8080/api/chirps \
  -H "Authorization: Bearer <access-token>" \
  -H "Content-Type: application/json" \
  -d '{"body":"hello chirpy"}'

# list chirps
curl -s "http://localhost:8080/api/chirps?sort=desc"
```

Optional query params for `GET /api/chirps`:

- `author_id=<uuid>`
- `sort=asc|desc`

### Refresh/revoke token

```bash
# Bearer = refresh token
curl -s -X POST http://localhost:8080/api/refresh \
  -H "Authorization: Bearer <refresh-token>"

curl -i -X POST http://localhost:8080/api/revoke \
  -H "Authorization: Bearer <refresh-token>"
```

### Webhook (Polka)

```bash
curl -i -X POST http://localhost:8080/api/polka/webhooks \
  -H "Authorization: ApiKey <POLKA_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"event":"user.upgraded","data":{"user_id":"<user-uuid>"}}'
```

This marks the user as `is_chirpy_red=true`.
